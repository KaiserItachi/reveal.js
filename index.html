GraphQL

What is GraphQL?

- GraphQL is a query language for APIs and a runtime for fulfilling those queries with your existing data.

History

  • Created by facebook in 2012.
  • Open Sourced in 2015
  • Stable Release in October 2016

Challenges in REST

  • To fetch connected data multiple calls required
  • Any changes in UI will inturn result in creation of new endpoint in REST Api making it a maintenance nightmare
  • Overfetching of data and wasteage of bandwitdth

GraphQL

  • One endpoint (/graphql)
  • Multiple resource retrieved from One call
  • Only retrieve data that is needed.

Example

Features of GraphQL

  • Strongly types objects and fields
  • Highly customizable queries
  • Nested/Parallel objects in a single request
  • Data aggregation from multiple sources
  • Integrated GUI with auto-completion, error check and documentation

Disadvantages of GraphQL

  • Security Risk: DoS with Single Query
  • No Support for file uploads
  • Caching at network level (Post to /graphql)
  • Unnecessary complexity when the data is not connected

Security

  • Timeouts: Can set the timeouts for query execution
  • Query Depth: Can set limitations on the depth of the Query
  • Query complexity: Can set limitations on query complexity

Future

  • Subscriptions - Looks like a query. Gives updates when something changes
  • Deffered Queries - Helps in showing something as fast as possible.
